# 上海幼升小择校助手 — Codex Agent 指令

## 项目概述

这是一个帮助上海家长进行幼升小择校决策的产品，当前版本 v8.0.1。
- **线上地址**：https://kallenhu1-spec.github.io/school-advisor/
- **主文件**：`index.html`（单文件架构，含全部前端逻辑+数据）
- **学校数据**：170+ 所学校，覆盖上海 9 个区
- **技术栈**：HTML + CSS + JavaScript + Chart.js（内联）

## 数据质量原则（所有 Agent 必须遵守）

> ⚠️ 这个产品帮助真实家长做择校决策，数据诚实是底线。

- **口碑数据（`var PR=`）大部分是 AI 生成的草稿，尚未经真实来源核实**，前端展示时必须附带免责说明
- **中签率数据**必须标注年份（`rateYear`）和是否为估算值（`rateEstimated`）
- 所有 Agent 在涉及数据修改时，必须同时更新 `dataSource` 来源字段
- 来源级别：`official`（官方）> `verified`（已核实）> `community`（社区）> `ai-draft`（AI草稿）
- 禁止把 `ai-draft` 数据当成真实数据展示，必须有 ⚠️ 标注

## 核心架构约束

1. **编码规范**：HTML 内 JS 中的中文必须直接写 UTF-8 字符串，**禁止** `unicode_escape` 或 `&#XXXXX;` 数字实体
2. **版本管理**：每次修改前，先将当前 `index.html` 复制到 `versions/vX.Y.Z-YYYYMMDD.html`
3. **版本号**：同步更新 `VERSION.json`
4. **数据结构**：学校数据在 `var SD=`，口碑在 `var PR=`，梯队在 `var TF=`，区域映射在 `var DN=`
5. **手机适配**：桌面用 `#school-table`，手机用 `#school-cards`，通过 `@media (max-width: 768px)` 切换

## GSTACK 多角色迭代框架

本项目采用 Garry Tan 的 gstack 多角色审查框架进行自动化迭代。每次迭代包含 4 个角色的审查：

### 迭代流程（自动执行）

```
1. CEO Agent      → 审查产品方向、商业化、用户价值
2. Design Agent   → 审查 UX/UI、情感化设计、AI Slop 检测
3. Engineering Agent → 审查代码质量、架构、性能
4. QA Agent       → 测试、风险检查、数据验证
```

### 迭代输出规范

每个 Agent 的输出必须包含：
- **发现的问题**（按优先级排序）
- **具体修改建议**（含代码示例）
- **预估影响**（用户体验/性能/商业价值）

最终由 Engineering Agent 汇总所有建议，生成一个 PR，等待人工 approve。

## 商业目标（Agent 决策参考）

- **核心用户**：上海幼升小家长（每年约 18 万+家庭）
- **商业模型**：免费工具 + 付费 AI 咨询服务
- **当前优先级**：
  1. 1v1 AI Chat 上线并可用
  2. 付费引导流程
  3. 微信生态融入
  4. 性能优化（index.html 体积过大）

## 当前定价策略（会变，以此处为准）

> ⚠️ 所有 Agent 在提及定价时，必须读取本节，不得使用自己「记忆」中的数字。

| 产品 | 当前定价 | 备注 |
|------|---------|------|
| AI 咨询次数包 | 待定（验证阶段可免费） | 上线后根据转化率调整 |
| 择校报告（付费生成） | 待定 | 核心付费产品，定价需测试 |

**定价原则**（比具体数字更重要）：
- 单价需低于用户「一顿饭钱」心理门槛
- 首次付费要有明确的价值感知（看到报告才付款，而非先付款）
- AI 成本极低（单次对话 < ¥0.01），毛利率目标 > 95%

## 文件变更规则

- 修改 `index.html` 前必须先存档到 `versions/`
- PR 标题格式：`[GSTACK-{角色}] {简述}`，如 `[GSTACK-ENG] 优化 Chart.js 懒加载`
- commit message 用中文，格式：`{类型}({范围}): {描述}`
- 不要修改 `.github/workflows/sync_to_cloudflare_repo.yml`
